# PR Summary Template

Use this structure when summarizing a change.

```md
## Summary

What changed and why?

## Files changed

- `path/to/file`: reason
- `path/to/file`: reason

## Behavior changed

Describe user-visible or system-visible behavior.

## Tests

Commands run:

```bash
<paste commands>
```

Results:

```text
<paste result summary>
```

## Security impact

Describe authentication, authorization, secrets, input validation, logging, or data exposure impact.

## Data impact

Describe migrations, data writes, backfills, transactions, or compatibility concerns.

## Risk

What could break?

## Follow-up

What remains to do?
```
